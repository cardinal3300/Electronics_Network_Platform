# Electronics_Network_Platform

**Веб-приложение на Django для управления сетью продажи электроники.**

Проект реализует иерархическую структуру поставщиков и точек продаж и предоставляет REST API.

---

## Стек технологий

- Python
- Django
- Django REST Framework
- PostgreSQL
- Django Filters
- Swagger (drf-yasg)

---

## Структура сети

Сеть состоит из трёх уровней:

1. Завод
2. Розничная сеть
3. Индивидуальный предприниматель

Иерархия строится через ссылку на поставщика.

---

## Модель сети

Каждое звено сети содержит:

* название

* контакты

    * email
    
    * страна
    
    * город
    
    * улица
    
    * номер дома

* продукты

* поставщик

* задолженность перед поставщиком

* дату создания

---

## Иерархия

Иерархия реализована через self-reference ForeignKey.

```ardeno
    provider = models.ForeignKey('self',
                                on_delete=models.SET_NULL,
                                null=True,
                                blank=True,
                                related_name='clients',
                                verbose_name='Поставщик')
```

---

## Валидация иерархии

Реализованы проверки:

1. объект не может ссылаться сам на себя

2. запрещены циклические зависимости

3. поставщик должен находиться выше по уровню

---

## Админ панель

В админке реализовано:

* просмотр объектов сети

* ссылка на поставщика

* фильтр по городу

* admin action очистки задолженности

**Очистка задолженности**

Можно выбрать несколько объектов и применить действие:
```Очистить задолженность перед поставщиком```

Все выбранные объекты получат `debt = 0`.

## API

Реализован CRUD для модели сети.

Примеры запросов:

```aredno
    GET /api/network/
    POST /api/network/
    PUT /api/network/{id}
    DELETE /api/network/{id}
```

---

## Фильтрация

Фильтрация объектов по стране:

```GET /api/network/?country=USA```

---

## Права доступа

Доступ к API имеют только:

* авторизованные пользователи

* активные сотрудники (`is_staff=True`)

## Установка

1. Клонировать репозиторий

```git clone ...```

2. Установить зависимости

```pip install -r requirements.txt```

3. Применить миграции

```python manage.py migrate```

4. Запустить сервер

```python manage.py runserver```
